## Phase Details

### Phase 1: Foundation
**Goal**: A working IRC bot that connects safely to Undernet, authenticates with X, manages its own reconnection, and has crash-safe score storage ready for the game engine.
**Depends on**: Nothing
**Requirements**: IRC-01, IRC-02, IRC-03, IRC-04, IRC-05, IRC-06, BOT-03, BOT-04, AUTH-01, AUTH-02, AUTH-03, SCORE-01, SCORE-04, CONF-01, CONF-02
**Success Criteria** (what must be TRUE):
  1. Bot connects to Undernet, logs in with X LoC credentials from config, and joins the configured default channels
  2. Bot automatically reconnects with exponential backoff after a forced disconnect; all channel game states reset to IDLE on disconnect with no orphaned timers
  3. All outbound IRC messages are queued through a rate-limited sender — burst output does not cause a flood disconnect
  4. Commands from a user whose nick matches an owner but whose hostmask does not match are rejected silently
  5. A score written to SQLite immediately before a simulated crash is present and uncorrupted on restart (WAL + transaction)

**UI hint**: no

### Phase 2: Game Engine + AI
**Goal**: A complete trivia game loop — `!start` to `!stop`, Claude-generated questions with pre-generated answer variants, race answer detection, per-round scoring, and leaderboard — running independently per channel.
**Depends on**: Phase 1
**Requirements**: GAME-01, GAME-02, GAME-03, GAME-04, GAME-05, GAME-06, GAME-07, GAME-08, GAME-09, AI-01, AI-02, AI-03, AI-04, AI-05, AI-06, SCORE-02, SCORE-03
**Success Criteria** (what must be TRUE):
  1. `!start` begins a game in the current channel; a trivia question generated by Claude appears with a countdown timer
  2. The first player to type a correct answer (case-insensitive, whitespace-stripped, Levenshtein-fuzzy) wins the point and the bot announces the winner; subsequent answers for the same question are dropped
  3. Two channels can run simultaneous games with fully independent state — a `!stop` in one channel has no effect on the other
  4. After the configured number of questions, the game ends and the bot announces final scores in-channel
  5. `!scores` shows current game scores and `!leaderboard` shows all-time top scores for that channel without requiring an active game
**Plans**: TBD
**UI hint**: no

### Phase 3: Owner Commands + Polish
**Goal**: Full owner control over the bot from IRC — channel join/part, topic and difficulty changes, config-at-game-start reload — plus CTCP handling and question length safety.
**Depends on**: Phase 2
**Requirements**: BOT-01, BOT-02, CONF-03, CONF-04, CONF-05
**Success Criteria** (what must be TRUE):
  1. Owner can make the bot join or part a channel mid-session with `!join #channel` / `!part #channel`; non-owners issuing these commands get no response
  2. Owner can change the active topic with `!topic <topic>` and the next question generated uses the new topic without restarting the bot
  3. Owner can change difficulty with `!difficulty <easy|medium|hard>` and the next question reflects the new difficulty level
  4. Editing the YAML config file while the bot is running and then issuing `!start` picks up the updated values without a restart
